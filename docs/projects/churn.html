Customer Churn Prediction

A Business' ability to accurately forecast future sales and the likelihood for customers to make return purchases is vital to any organization. With Machine learning we are able to predict when a customer is likely to return vs. churn and go to our competitors.This project consists of 2 parts. Part 1: An inital EDA and model building. Part 2: Feature Enginering and Model Fine Tuning

Sections

Data Cleaning & EDA

For this project we will be building a binary classification model to identify customers that churn. For this we need our data to be able to be fitted on to a regression model thus we need to explore the distribution of the data. We also need to detect any missing values and outliers that may impact our model's ability to predict

  • Tools: Pandas, Scipy & Seaborn
  • View Code

Initial Model Building & Testing

Python includes an extensive library of ready-made models that we can utilize for this purpose. The key objective is to select the one that performs best for our data. During the EDA & data cleaning we found our data to be missing a lot of values thus our model needs to be robust enough to handle this type of data

  • Tools: Sci-kit Learn, XGBoost & TensorFlow
  • View Code

Feature Engineering

In the initial model building phase we found the best performing ML model. For us to make improvements to the accuracy of our model we need to process the data so that it gives our machine the ability to detect trends in our data and make more accurate predictions

  • Tools: Pandas, Scipy & Seaborn
  • View Code

Machine Learning Model Final

The final and most crucial step of building machine learning applications is to fine tune the hyperparameter of our model. The parameters we will be setting will help fit our model to the variance better and ensure that we don't create overfitting to the training data that we will use

  • Tools: XGBoost, CATBoost & LightGB
